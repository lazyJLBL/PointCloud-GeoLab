# v1.0.0 Portfolio-Stable Release

PointCloud-GeoLab v1.0.0 is the portfolio-stable and reviewer-stable formal
release. It focuses on reproducible review workflows, bounded API and CLI
contracts, artifact validation, and documented real-data-format workflows.

This release is not a production LiDAR stack, not an Open3D/PCL replacement,
not a full nonlinear GICP optimizer, not an official KITTI benchmark, not a
SLAM backend, not CUDA accelerated, and not a PointNet training release.

## Highlights

- Current package metadata is prepared for `1.0.0` while v0.1.0 and v0.1.1
  remain historical releases.
- User-provided KITTI-like LiDAR segmentation workflow emits Markdown, HTML,
  JSON metrics, timing, memory metadata, cluster reports, PLY, and PNG figures.
- `verify-realdata` runs a synthetic KITTI-like dry-run in CI and verifies the
  same artifact schema used for local user-provided frames.
- Scale benchmark writes JSON, CSV, Markdown, and PNG with repeat
  mean/std/min/max timing fields, peak memory metadata, and machine info.
- API and CLI stability docs define stable, experimental, deprecated, and
  optional surfaces for reviewers.
- Gallery documentation collects synthetic, tiny fixture, and user-provided
  workflow visuals with clear labels.
- Limitations and project-boundary docs keep GICP-style ICP experimental and
  keep issue #1 and issue #2 as scoped roadmap items.

## Known Limits

- Synthetic demos and tiny fixtures are smoke checks, not real-data benchmark
  results.
- The KITTI workflow handles user-provided KITTI-like single frames; it is not
  an official KITTI benchmark.
- GICP remains GICP-style covariance-weighted ICP and is not a full nonlinear
  GICP optimizer.
- The repository is not a SLAM backend, not CUDA accelerated, not a PointNet
  training release, and not a PCL/Open3D replacement.
- Full 1M-point scale benchmark runs are intended for manual gates, not the
  default CI path.
- Issue #1 remains open; v1.0.0 is not a full nonlinear GICP optimizer.

## Open Roadmap Items

- Keep issue #1 as future nonlinear covariance-ICP optimizer work.
- Treat official real KITTI benchmark reporting as a separate future task from

- Continue improving API and CLI contracts through small compatibility-focused
