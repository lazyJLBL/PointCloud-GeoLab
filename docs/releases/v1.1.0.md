# v1.1.0 Experimental Web Console MVP

PointCloud-GeoLab v1.1.0 adds an experimental reviewer-oriented Web Console
on top of the stable Python task API. The core algorithm package, CLI, and
stable public API remain intact.

This release is not a production web platform, not a production LiDAR stack,
not an Open3D/PCL replacement, not a full nonlinear GICP optimizer, not an
official KITTI benchmark, not a SLAM backend, not CUDA accelerated, and not a
PointNet training release.

## Highlights

- Package, citation, README, release checks, and Web package metadata are set
  to `1.1.0` while v1.0.0 remains a historical portfolio-stable release.
- Experimental FastAPI backend exposes dataset upload, preview, task, and
  artifact routes under `/api`.
- Experimental Vue 3 frontend provides Dashboard, Datasets, Preprocessing,
  Registration, Segmentation, Geometry, Primitives, Benchmark, Portfolio, and
  Reports pages.
- Web uploads accept `.ply`, `.pcd`, `.xyz`, `.txt`, `.bin`, and `.off` files,
  with size limits and path traversal checks.
- Web artifact downloads now preserve nested relative paths and reject
  traversal attempts.
- Web benchmark tasks default to quick mode and reject overly long repeat,
  query, and point-count settings.
- `make verify-web` runs backend Web tests and the frontend production build.
- Manual release-gate workflow now includes Web verification without publishing
  artifacts.

## Known Limits

- The Web Console is experimental and reviewer-oriented, not a production web
  platform.
- Web tasks are currently synchronous. Long portfolio or benchmark requests may
  block until completion.
- Benchmark timing and memory values are local machine references only.
- Synthetic demos and tiny fixtures are smoke checks, not real-data benchmark
  results.
- The KITTI workflow handles user-provided KITTI-like single frames; it is not
  an official KITTI benchmark.
- GICP remains GICP-style covariance-weighted ICP and is not a full nonlinear
  GICP optimizer.
- The repository is not a SLAM backend, not CUDA accelerated, not a PointNet
  training release, and not a PCL/Open3D replacement.
- Frontend builds may report Vite/Rollup bundle-size warnings.
