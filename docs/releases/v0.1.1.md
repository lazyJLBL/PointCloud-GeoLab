# v0.1.1 Hardening Release

PointCloud-GeoLab v0.1.1 is a reviewer-oriented portfolio and learning release
that tightens verification, packaging sanity, benchmark metadata, and
documentation boundaries without adding large new algorithms.

## Highlights

- Repository hygiene checks for tracked generated outputs, local documentation
  links, overclaim wording, text formatting, and version metadata.
- `verify-core` includes hygiene, DevContainer, packaging, and tiny dataset
  fixture checks while staying dependency-light.
- Benchmark CLI supports local repeat statistics and lightweight `tracemalloc`
  memory metadata.
- Portfolio pipeline writes both `report.md` and a simple static `report.html`.
- DevContainer files document a reproducible reviewer environment without CUDA,
  large ML runtimes, or real dataset downloads.
- Tiny synthetic KITTI-like `.bin` and ModelNet-like `.off` fixtures validate
  file-format readers and SHA256 manifest checks in CI.
- Release checklist, reviewer checklist, coverage notes, and project-boundary
  docs describe what is implemented and what remains future work.

## Verification

```bash
python -m pip install -e ".[dev,vis,bench]"
make verify-core
make verify-portfolio
make verify-benchmarks
make verify-release-candidate
```

On Windows without `make`, use `mingw32-make` for the same targets.

## Generated Artifacts

The release expects reviewers to generate portfolio and benchmark artifacts
locally. These paths are ignored by Git and should not be committed:

- `outputs/portfolio_demo/report.md`
- `outputs/portfolio_demo/report.html`
- `outputs/portfolio_demo/metrics.json`
- `outputs/portfolio_demo/figures/*.png`
- `outputs/portfolio_demo/artifacts/transformation.json`
- `outputs/benchmarks/*.csv`
- `outputs/benchmarks/*.json`
- `outputs/benchmarks/*.md`
- `outputs/benchmarks/*.png`

The machine-readable companion manifest is
[`v0.1.1_artifacts.json`](v0.1.1_artifacts.json).

## Known Limits

- Synthetic demos and tiny fixtures are smoke evidence, not real-data
  benchmark results.
- The KITTI-like fixture is not a real KITTI frame and does not close the real
  KITTI benchmark roadmap item.
- GICP remains GICP-style covariance-weighted ICP. It is not a full nonlinear
  GICP optimizer.
- The repository is not a SLAM backend, CUDA stack, PointNet training release,
  or PCL/Open3D replacement.
- Optional Open3D, Plotly, laspy, PyTorch, SciPy, scikit-learn, and pandas
  paths remain optional comparisons or demos.

## Open Roadmap Items

- The project still does not include a full nonlinear GICP optimizer; keep that
  as a separate future algorithm task.
- Add a real KITTI benchmark report using user-provided data under
  `data/external/`.
- Expand reproducible environment docs only as needed; do not publish images or
  releases automatically.
